Q1 Explain the different types of control structures in Java with examples. 7 Marks
Control structures are statements that control the flow of execution in a program. Java has three main categories.
1. Sequential Structure
  • Statements execute line by line, top to bottom. This is the default flow.
2. Selection / Decision-Making Structures
  • if statement — executes a block if condition is true.
  • if-else — executes one of two blocks based on condition.
  • else-if ladder — tests multiple conditions in sequence.
  • switch-case — selects one block from many based on a value.
3. Iteration / Loop Structures
  • for loop — used when number of iterations is known.
  • while loop — used when iterations depend on a condition (entry-controlled).
  • do-while loop — executes at least once (exit-controlled).
4. Jump Statements
  • break — exits the loop or switch.
  • continue — skips current iteration.
  • return — exits from a method.
Example Program
public class ControlDemo {
    public static void main(String[] args) {

        // if-else
        int x = 10;
        if (x > 5)
            System.out.println("x is greater than 5");
        else
            System.out.println("x is not greater than 5");

        // for loop
        for (int i = 1; i <= 5; i++)
            System.out.println("i = " + i);

        // switch-case
        int day = 3;
        switch(day) {
            case 1: System.out.println("Monday"); break;
            case 3: System.out.println("Wednesday"); break;
            default: System.out.println("Other day");
        }
    }
}
Output: x is greater than 5 → i = 1 to 5 → Wednesday
Q2 List and explain the features of Java. 7 Marks
Java is a high-level, object-oriented, platform-independent programming language developed by Sun Microsystems in 1995. Its key features are:
  • Simple — Easy syntax, no pointers, automatic garbage collection.
  • Object-Oriented — Everything is an object. Supports encapsulation, inheritance, polymorphism, abstraction.
  • Platform Independent — Java code is compiled to bytecode which runs on any JVM regardless of OS.
  • Secure — No explicit pointers, bytecode verification, SecurityManager class.
  • Robust — Strong type checking, exception handling, memory management via GC.
  • Multithreaded — Built-in support for concurrent execution using Thread class.
  • Distributed — Supports networking via java.net package; RMI and CORBA support.
  • Dynamic — Classes loaded at runtime; supports reflection.
  • Portable — Same bytecode runs on any platform — "Write Once, Run Anywhere."
  • High Performance — JIT (Just-In-Time) compiler improves execution speed.
  • Interpreted — JVM interprets bytecode line by line.
  • Architecture Neutral — Primitive data type sizes are fixed regardless of platform.
Exam tip: Remember the acronym S-O-P-S-R-M-D-D-P-H-I-A for all 12 features.
Q3 What is bytecode? Justify why Java is platform independent. 7 Marks
Bytecode is the intermediate, machine-neutral code generated by the Java compiler. It is stored in .class files and is not OS-specific.
Compilation Process
Source Code (.java) │ ▼ javac (Java Compiler) Bytecode (.class) │ ▼ JVM on any OS Machine Code → Execution
Why Platform Independent?
  • Traditional languages (C, C++) compile directly to platform-specific machine code.
  • Java compiles to bytecode — a universal instruction set understood by the JVM.
  • The JVM is platform-dependent but the bytecode is not. JVM is available for Windows, Linux, macOS, etc.
  • The same .class file runs on any OS with a JVM — enabling "Write Once, Run Anywhere".
  • Primitive data type sizes are also fixed (e.g., int = 32 bits always), ensuring consistent behavior.
Key distinction: Java is platform-independent; JVM is platform-dependent.
Q4 What is method overloading? Explain with an example. 7 Marks
Method Overloading is defining multiple methods with the same name but different parameter lists (type, number, or order). It is a compile-time (static) polymorphism.
Rules for Overloading
  • Methods must differ in number of parameters, type of parameters, or order of parameters.
  • Return type alone cannot differentiate overloaded methods.
  • Resolved at compile time — also called early binding.
public class Calculator {

    // Two integers
    int add(int a, int b) {
        return a + b;
    }

    // Three integers
    int add(int a, int b, int c) {
        return a + b + c;
    }

    // Double parameters
    double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        Calculator c = new Calculator();
        System.out.println(c.add(2, 3));          // 5
        System.out.println(c.add(2, 3, 4));       // 9
        System.out.println(c.add(2.5, 3.5));      // 6.0
    }
}
Advantage: Improves code readability and reusability under one meaningful method name.

Q6 Explain the types of constructors with the help of an example. 7 Marks
A constructor is a special method that has the same name as the class, no return type, and is called automatically when an object is created to initialize it.
Types of Constructors
  • Default Constructor — No parameters. Provided by JVM if none is defined. Sets defaults.
  • Parameterized Constructor — Accepts arguments to initialize object with specific values.
  • Copy Constructor — Takes an object of the same class and creates a copy of it.
class Student {
    String name;
    int age;

    // Default Constructor
    Student() {
        name = "Unknown";
        age = 0;
    }

    // Parameterized Constructor
    Student(String n, int a) {
        name = n;
        age = a;
    }

    // Copy Constructor
    Student(Student s) {
        name = s.name;
        age = s.age;
    }

    void display() {
        System.out.println(name + " - " + age);
    }

    public static void main(String[] args) {
        Student s1 = new Student();          // Default
        Student s2 = new Student("Alice", 20); // Parameterized
        Student s3 = new Student(s2);       // Copy
        s1.display(); s2.display(); s3.display();
    }
}
Output: Unknown - 0 → Alice - 20 → Alice - 20

Q8 Write a Java program to demonstrate the use of the 'this' reference. 7 Marks
The 'this' keyword refers to the current object of a class. It is used to differentiate instance variables from local variables and to call other constructors.
Uses of 'this'
  • Resolve ambiguity between instance variables and parameters of the same name.
  • this() — call another constructor of the same class (constructor chaining).
  • Pass the current object as a parameter to a method.
  • Return the current object from a method.
class Employee {
    String name;
    int id;

    Employee(String name, int id) {
        this.name = name;  // this.name = instance, name = parameter
        this.id = id;
    }

    // Constructor chaining using this()
    Employee() {
        this("Default", 0);
    }

    Employee getObject() {
        return this;  // return current object
    }

    void display() {
        System.out.println("ID: " + id + ", Name: " + name);
    }

    public static void main(String[] args) {
        Employee e1 = new Employee("Raj", 101);
        e1.display();
        Employee e2 = new Employee();
        e2.display();
    }
}
Output: ID: 101, Name: Raj → ID: 0, Name: Default
Q9 What is polymorphism? Explain with an example. 7 Marks
Polymorphism means "many forms." In Java, the same method or object can behave differently based on the context. It comes from the Greek words poly (many) + morph (forms).
Types of Polymorphism

class Animal {
    void sound() { System.out.println("Animal makes sound"); }
}
class Dog extends Animal {
    @Override
    void sound() { System.out.println("Dog barks"); }
}
class Cat extends Animal {
    @Override
    void sound() { System.out.println("Cat meows"); }

Q13 Explain exception handling in Java using try-catch-finally with an example. 7 Marks
An exception is an abnormal event that disrupts normal program flow. Java provides a structured mechanism using try, catch, finally, throw, and throws.
Keywords Explained
  • try — block of code that might throw an exception.
  • catch — handles the specific exception thrown in try.
  • finally — always executes (with or without exception) — used for cleanup.
  • throw — manually throws an exception.
  • throws — declares exceptions a method may throw.
public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int a = 10, b = 0;
            int result = a / b;  // throws ArithmeticException
            System.out.println("Result: " + result);
        }
        catch (ArithmeticException e) {
            System.out.println("Caught: " + e.getMessage());
        }
        catch (Exception e) {
            System.out.println("General Exception: " + e.getMessage());
        }
        finally {
            System.out.println("Finally block always runs");
        }
        System.out.println("Program continues...");
    }
}
Output:
Caught: / by zero
Finally block always runs
Program continues...
Exception Hierarchy
Throwable ├── Error (OutOfMemoryError, StackOverflowError) └── Exception ├── Checked (IOException, SQLException) └── Unchecked / RuntimeException (NullPointer, ArrayIndexOutOfBounds)

Q17 Describe an abstract class Shape with Triangle, Rectangle, Circle subclasses overriding area(). 7 Marks
This question demonstrates abstraction — defining a method contract in the parent and forcing subclasses to provide specific implementations.
abstract class Shape {
    abstract double area();  // abstract method - no body
    void display() {
        System.out.println("Area = " + area());
    }
}

class Triangle extends Shape {
    double base, height;
    Triangle(double b, double h) { base=b; height=h; }

    @Override
    double area() { return 0.5 * base * height; }
}

class Rectangle extends Shape {
    double length, width;
    Rectangle(double l, double w) { length=l; width=w; }

    @Override
    double area() { return length * width; }
}

class Circle extends Shape {
    double radius;
    Circle(double r) { radius=r; }

    @Override
    double area() { return Math.PI * radius * radius; }
}

public class ShapeTest {
    public static void main(String[] args) {
        Shape[] shapes = {
            new Triangle(6, 4),
            new Rectangle(5, 3),
            new Circle(7)
        };
        for (Shape s : shapes)
            s.display();
    }
}

Q20 What is polymorphism? Explain dynamic binding with an example. 7 Marks
Dynamic binding (late binding) is the process where the method to be called is determined at runtime based on the actual object type, not the reference type.
  • Achieved through method overriding in Java.
  • The reference type is the parent class; the object type is the child class.
  • JVM uses a virtual method table (vtable) to resolve the correct method at runtime.
  • Enables runtime polymorphism — core of OOP's flexibility.
class Bank {
    float interestRate() { return 0; }
}

class SBI extends Bank {
    @Override
    float interestRate() { return 8.4f; }
}

class ICICI extends Bank {
    @Override
    float interestRate() { return 7.5f; }
}

public class BindingDemo {
    public static void main(String[] args) {
        Bank b;

        b = new SBI();
        System.out.println("SBI Rate: " + b.interestRate());  // 8.4

        b = new ICICI();
        System.out.println("ICICI Rate: " + b.interestRate()); // 7.5
    }
}
Though the reference is Bank, the method called is determined by the actual object at runtime — this is dynamic binding.

Unit 6 — Multithreading
Q22 What is Multithreading? Explain Thread life cycle with proper Java code. 7 Marks
Multithreading is the ability to execute multiple threads simultaneously. A thread is the smallest unit of processing. Multithreading improves CPU utilization and application responsiveness.
Thread Life Cycle States
NEW ──start()──→ RUNNABLE ──selected by JVM──→ RUNNING │ BLOCKED/WAITING ←── wait()/sleep() │ notify()/time up │ RUNNABLE │ run() ends │ TERMINATED
  • New — Thread object created but start() not called.
  • Runnable — start() called, waiting for CPU.
  • Running — Thread is executing.
  • Blocked/Waiting — Waiting for I/O, lock, or sleep().
  • Terminated (Dead) — run() method has completed.
class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 1; i <= 3; i++) {
            System.out.println(Thread.currentThread().getName()
                               + " - Count: " + i);
            try { Thread.sleep(500); }
            catch (InterruptedException e) { e.printStackTrace(); }
        }
    }
}

public class ThreadDemo {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();
        MyThread t2 = new MyThread();
        t1.setName("Thread-A");
        t2.setName("Thread-B");
        t1.start();  // New → Runnable
        t2.start();
    }
}
Two ways to create threads: (1) extend Thread class, (2) implement Runnable interface. Runnable is preferred as Java supports only single inheritance.
Q23 Explain the thread life cycle in Java with a suitable diagram. 7 Marks
The Thread Life Cycle in Java consists of 5 states managed by the Thread Scheduler of the JVM.
┌─────────────────────────────────────────────────────┐ │ THREAD LIFE CYCLE │ ├─────────────────────────────────────────────────────┤ │ │ │ [ NEW ] ──── start() ────→ [ RUNNABLE ] │ │ │ │ │ scheduler picks │ │ │ ▼ │ │ [ RUNNING ] │ │ / \ │ │ sleep/wait() run() ends │ │ / \ │ │ [ BLOCKED/WAITING ] [ TERMINATED ] │ │ │ │ │ notify/time up │ │ │ │ │ [ RUNNABLE ] ────────────────── │ └─────────────────────────────────────────────────────┘
State Details
  • New — Thread t = new Thread(). Object created, not started.
  • Runnable — After t.start(). Thread is ready; waiting in the ready queue.
  • Running — CPU is assigned. run() is executing.
  • Blocked/Waiting — Thread paused due to sleep(ms), wait(), or I/O. Returns to Runnable after condition is met.
  • Terminated/Dead — run() finishes or stop() is called (deprecated). Thread cannot be restarted.

Q25 What is inheritance? Explain all types with an example. 7 Marks
Inheritance allows a class (child/subclass) to acquire properties and behaviors of another class (parent/superclass). It promotes code reusability and establishes an IS-A relationship.
Types of Inheritance
1. Single A → B 2. Multilevel A → B → C 3. Hierarchical A → B, A → C 4. Multiple A, B → C (via Interfaces only in Java) 5. Hybrid Combination of above (via Interfaces) 
// 1. Single Inheritance
class Animal { void eat() { System.out.println("Eating"); } }
class Dog extends Animal { void bark() { System.out.println("Barking"); } }

// 2. Multilevel Inheritance
class A { void showA() { System.out.println("A"); } }
class B extends A { void showB() { System.out.println("B"); } }
class C extends B { void showC() { System.out.println("C"); } }

// 3. Hierarchical Inheritance
class Vehicle { void move() { System.out.println("Moving"); } }
class Car  extends Vehicle { }
class Bike extends Vehicle { }

// 4. Multiple (via Interface)
interface Flyable   { void fly(); }
interface Swimmable { void swim(); }
class Duck implements Flyable, Swimmable {
    public void fly()  { System.out.println("Flying"); }
    public void swim() { System.out.println("Swimming"); }
}

public class InheritanceTest {
    public static void main(String[] args) {
        C obj = new C();
        obj.showA(); obj.showB(); obj.showC();
        new Duck().fly();
    }
}
Note: Java does NOT support multiple inheritance via classes (to avoid the Diamond Problem). It is achieved using interfaces.
Q26 Explain thread synchronization with an example / Write a program that uses threads. 7 Marks
Thread Synchronization is the process of controlling access to shared resources so that only one thread at a time can access a critical section. This prevents race conditions and ensures data consistency.
Why Synchronization?
  • Without synchronization, multiple threads may read/write a shared variable simultaneously causing incorrect results.
  • Java provides the synchronized keyword to lock a method or block for exclusive access.
  • Only one thread can hold the monitor/lock of an object at a time.
Without Sync (Problem)
class Counter {
    int count = 0;
    void increment() { count++; }  // not thread-safe!
}
With Synchronized Method (Solution)
class PrintTable {
    synchronized void printTable(int n) {
        for (int i = 1; i <= 5; i++) {
            System.out.println(n + " × " + i + " = " + (n * i));
            try { Thread.sleep(200); }
            catch (InterruptedException e) {}
        }
    }
}

class T1 extends Thread {
    PrintTable pt;
    T1(PrintTable pt) { this.pt = pt; }
    public void run() { pt.printTable(5); }
}
class T2 extends Thread {
    PrintTable pt;
    T2(PrintTable pt) { this.pt = pt; }
    public void run() { pt.printTable(10); }
}

public class SyncDemo {
    public static void main(String[] args) {
        PrintTable obj = new PrintTable();
        T1 t1 = new T1(obj);
        T2 t2 = new T2(obj);
        t1.start();
        t2.start();
    }
}
Without synchronized, output of table 5 and table 10 would be interleaved. With it, T1 completes fully before T2 starts — output is clean and ordered.
Synchronized Block (alternative)
void increment() {
    synchronized(this) {
        count++;
    }
}
Key Concepts: Monitor lock · Race condition · Critical section · Mutual exclusion (Mutex)
